从网站URL中删除www

使用这样或那样的方法会影响SEO吗？

你可能想知道使用其中一个是否会对你的搜索引擎优化产生影响。答案是:不。这真的只是一个偏好/美学的问题。只要确保你在谷歌搜索控制台中正确地添加了www和非www域，正如这里的所描述的，以确保谷歌能够正确地索引你的网站。

从您的域名中删除www

RewriteEngine On
RewriteCond %{HTTP_HOST} ^www.example.com$ [NC]
RewriteRule ^(.*)$ http://example.com/$1 [R=301,L]

注意，Apache的mod_rewrite模块需要启用。否则上面的片段就不行了。

现在，在Nginx中，这个代码片段有一点不同，但是当放在适当的配置文件中时(这取决于您的设置)，会产生完全相同的结果:

server {
 server_name www.example.com;
 return 301 http://example.com$request_uri;
}

现在只需重启Nginx，您就可以开始工作了！

添加www而不是删除它

若要执行与上一节相反的操作，请将以下代码添加到您的。htaccess文件:

RewriteEngine On 
RewriteCond %{HTTP_HOST} ^example.com$ 
RewriteRule (.*) http://www.example.com$1 [R=301]

在Nginx中，只需要这样做:

server {
 server_name example.com;
 return 301 http://www.example.com$request_uri;
}

这就是全部了！

但是安全性呢？

正如你们中的一些人在评论中指出的(谢谢！)，当您决定使用非www类型的URL时，会有一些安全问题。

如果你在子域名上运行各种不同的服务，你就要冒在你的主非www主页和所述服务之间共享cookies的风险。如果某些敏感数据存储在您不想与第三方共享的cookies中，这可能是一件坏事。
