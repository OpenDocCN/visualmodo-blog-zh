# 最佳 API 架构实践服务器或客户端 API 调用？

什么是 API？API(应用程序编程接口)是一种软件接口，它规定了不同的应用程序如何通信和共享数据。每当你使用优步之类的应用程序，在 Twitter 上发送消息，甚至在智能手机上查看天气时，你都在使用 API。API 允许这些应用程序通过调用进行通信。它们(API)规定了发出的调用、收到的响应以及使用的约定和数据格式。在本文中，我们将看到服务器或客户端 API 调用的最佳 API 架构实践。

什么是客户机-服务器体系结构？我们今天使用的大多数应用程序都是基于这种模型的。在这里，用户设备使用位于中央的服务器网络进行通信并获取他们需要的所有信息。这意味着它们不直接相互通信，而是通过服务器进行通信。在此模型中，用户设备(智能手机、笔记本电脑和其他智能设备)被称为客户端。当用户使用这些设备访问应用程序时，他们会向服务器发送请求以获取所需的信息。作为回报，服务器会根据应用程序的需求做出响应。有些开发人员在构建他们的 API 时会实现仅服务器或仅客户端的架构。这两者的工作方式不同，对于所有开发人员来说，理解客户端和服务器端之间的差异非常重要。

仅客户端的 API 体系结构在设计 API 体系结构时，一些开发人员可能会发现自己处于不确定要实现哪种体系结构的情况。他们中的一些人倾向于仅客户端架构。但是这个架构是如何工作的呢？客户端用于表示最终用户可以看到的一切，例如图像、文本和整个用户界面，以及应用程序在用户设备上执行的每个操作。

因此，顶级 API 架构实践之一。例如，客户端使用浏览器来解释标记语言，如 [CSS 和 HTML](https://visualmodo.com/css-basic-tips-for-website-owners/) 。这迫使许多开发人员在客户端实现流程，而不是在服务器端完成所有事情。访问 youtube.com 可以看到客户端流程的一个很好的例子。主页由 YouTube 推荐的视频组成，基于不同方面，如一个人观看的视频。在这里，浏览器解释决定 YouTube 主页面外观的 JavaScript、CSS 和 HTML。这发生在客户端。
